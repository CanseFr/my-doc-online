# 1. Fondamentaux du développement logiciel

## Attentes principales

Un expert en ingénierie doit maîtriser **parfaitement les bases**. C’est non négociable.

### 1.1 Programmation

* Maîtrise d’un langage principal (Java, TypeScript, Python, C#, etc.)
* Compréhension de :

    * POO (héritage, polymorphisme, encapsulation)
    * Programmation fonctionnelle (map, filter, reduce, immutabilité)
    * Gestion des erreurs
    * Logging
    * Structures de données (liste, pile, file, map, set)
    * Complexité algorithmique (Big O)
* Savoir écrire du **code propre et lisible**

### 1.2 Clean Code

Primordial :

* Nommage clair
* Fonctions courtes
* Une fonction = une responsabilité
* Pas de duplication
* Code testable
* Respect des principes **SOLID**

### 1.3 Design Patterns (très attendu)

Il doit connaître et savoir quand utiliser :

* Factory
* Singleton
* Strategy
* Observer
* Repository
* Adapter
* MVC
* Dependency Injection

Pas juste les connaître → **savoir pourquoi on les utilise**.

# 2. Architecture logicielle

C’est là qu’on attend vraiment quelqu’un avec ce titre.

## 2.1 Comprendre les architectures

Il doit comprendre :

* Monolithe
* Architecture en couches
* Microservices
* Architecture hexagonale
* Clean Architecture
* Event-driven architecture
* SOA

Et surtout :

* Avantages / inconvénients
* Quand utiliser quoi
* Les problèmes que ça résout

## 2.2 Concepts d’architecture

Primordial :

* Couplage faible
* Forte cohésion
* Séparation des responsabilités
* Domain Driven Design (DDD)
* Scalabilité
* Résilience
* Maintenabilité
* Testabilité
* Observabilité

Il doit comprendre :

> Une architecture n’est pas là pour faire joli, mais pour résoudre des problèmes.

## 2.3 Modélisation

Il doit savoir faire :

* Diagramme UML (classe, séquence, composant)
* Modélisation BDD
* Modélisation API
* Schémas d’architecture (C4 model idéalement)

# 3. Backend / API

## Attentes

* Savoir créer une API REST propre
* Comprendre :

    * HTTP
    * REST
    * JSON
    * Authentification (JWT, session, OAuth)
    * Autorisation (roles, permissions)
    * Pagination
    * Filtrage
    * Upload de fichiers
    * Validation des données
    * Gestion des erreurs

### Très important :

* Sécurité API
* Documentation API (Swagger / OpenAPI)
* Versionning d’API

---

# 4. Base de données

## Attentes

* SQL (indispensable)
* Modélisation relationnelle
* Index
* Transactions
* Normalisation
* ORM (Prisma, Hibernate, TypeORM…)

Comprendre la différence :

* SQL vs NoSQL
* Quand utiliser MongoDB
* Quand utiliser PostgreSQL
* Quand utiliser Redis

# 6. Qualité logicielle

Primordial dans une grande entreprise.

## Attentes

* Tests unitaires
* Tests d’intégration
* Tests e2e
* TDD (connaître au moins le principe)
* Lint
* Formatage
* Revue de code

> On ne juge pas un développeur à son code, mais à la qualité globale de son projet.

# 7. Sécurité

Souvent négligé par les juniors → mais très attendu pour ce titre.

## Attentes minimales

* OWASP Top 10
* Injection SQL
* XSS
* CSRF
* Authentification sécurisée
* Hash de mot de passe (bcrypt)
* HTTPS
* Gestion des tokens
* CORS
* Permissions
* Secrets (ne jamais mettre dans le code)

# 9. Soft skills (très sous-estimé mais crucial)

Un architecte cherche quelqu’un qui :

* Pose des questions
* Documente
* Cherche à comprendre avant de coder
* Sait reconnaître quand il ne sait pas
* Est autonome
* Est curieux
* Est rigoureux
* Termine ce qu’il commence
* Pense en **système** et pas juste en code
