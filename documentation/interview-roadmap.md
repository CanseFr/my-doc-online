# Interview technique

## Objectif

Construire une expertise autonome dediee a la preparation des entretiens techniques a partir du fichier source `nouveau-fichier-a-traiter/interview-top-question.md`.

Le but de cette roadmap est simple :

- separer les ressources par langage ou framework
- garder une structure claire par niveau `debutant`, `avance`, `pro`, `expert`
- permettre une consultation rapide dans l'application sans melanger les thematiques

## Principe editorial

Chaque phase correspond a une techno precise et contient une seule bulle.

Dans chaque bulle, on retrouve toujours quatre groupes de connaissance :

- `Debutant`
- `Avance`
- `Pro`
- `Expert`

Ce choix permet de garder une lecture homogene dans l'application et d'eviter tout melange artificiel entre data, backend et frontend.

## Intention produit

Cette expertise doit rester pratique :

- orientee preparation entretien et revision rapide
- suffisamment structuree pour servir de base editoriale plus tard
- exploitable autant pour un profil junior que pour un profil senior
